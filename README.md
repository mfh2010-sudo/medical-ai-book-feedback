# صفحة مقترحات ومراجعات الكتاب
### الذكاء الاصطناعي في الممارسة الطبية

موقع بسيط يتيح لقراء الكتاب إرسال مقترحات وملاحظات (عامة أو مرتبطة بفصل معين)، مع لوحة مراجعة خاصة بك لمتابعة كل ما يصل.

---

## 📁 محتويات المشروع

```
feedback-app/
├── pages/
│   ├── index.js          ← الصفحة العامة (يراها القراء)
│   ├── admin.js           ← لوحة المراجعة (محمية بكلمة مرور)
│   └── _app.js
├── lib/
│   └── supabaseClient.js  ← اتصال قاعدة البيانات
├── styles/
│   ├── globals.css
│   └── admin.css
├── supabase_schema.sql    ← الكود الذي تُنفّذه في Supabase لإنشاء الجدول
├── .env.local.example     ← انسخه إلى .env.local وضع قيمك الحقيقية
└── package.json
```

## 🚀 خطوات الإعداد (مرة واحدة فقط)

### الخطوة 1 — إنشاء حساب Supabase مجاني

1. اذهب إلى [supabase.com](https://supabase.com) وأنشئ حساباً مجانياً
2. أنشئ مشروعاً جديداً (اختر أي اسم، مثلاً `kitab-feedback`)
3. انتظر دقيقتين حتى يجهز المشروع

### الخطوة 2 — إنشاء جدول البيانات

1. من القائمة الجانبية في Supabase، افتح **SQL Editor**
2. اضغط **New Query**
3. افتح ملف `supabase_schema.sql` المرفق، انسخ محتواه بالكامل، الصقه، ثم اضغط **Run**
4. تأكد من ظهور رسالة نجاح بدون أخطاء

### الخطوة 3 — جلب مفاتيح الاتصال

1. من القائمة الجانبية، اذهب إلى **Project Settings → API**
2. انسخ القيمتين:
   - **Project URL** → هذه قيمة `NEXT_PUBLIC_SUPABASE_URL`
   - **anon public key** → هذه قيمة `NEXT_PUBLIC_SUPABASE_ANON_KEY`

### الخطوة 4 — إعداد متغيرات البيئة محلياً (للتجربة قبل الرفع)

```bash
cp .env.local.example .env.local
```

ثم افتح `.env.local` وضع القيم الحقيقية التي نسختها، واختر كلمة مرور قوية لـ `NEXT_PUBLIC_ADMIN_PASSWORD`.

### الخطوة 5 — التجربة محلياً (اختياري)

```bash
npm install
npm run dev
```

افتح `http://localhost:3000` للصفحة العامة، و`http://localhost:3000/admin` للوحة المراجعة.

## ☁️ الرفع على Vercel

### الطريقة الأسهل — عبر GitHub

1. ارفع مجلد المشروع كاملاً (بدون `.env.local` — هو مُستثنى تلقائياً) إلى مستودع GitHub جديد
2. اذهب إلى [vercel.com](https://vercel.com) وسجّل دخول (يمكن عبر GitHub مباشرة)
3. اضغط **Add New Project** واختر المستودع الذي رفعته
4. **قبل الضغط على Deploy**، افتح قسم **Environment Variables** وأضف المتغيرات الثلاثة:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `NEXT_PUBLIC_ADMIN_PASSWORD`
5. اضغط **Deploy** وانتظر دقيقة تقريباً

بعد الانتهاء ستحصل على رابطين:
- `https://your-project.vercel.app` ← شاركه مع قراء الكتاب
- `https://your-project.vercel.app/admin` ← خاص بك فقط، احتفظ به سرياً

## 🔒 ملاحظة أمنية مهمة

كلمة مرور لوحة الإدارة الحالية بسيطة (مقارنة نصية في الواجهة الأمامية) — مناسبة لمشروع شخصي يستخدمه شخص واحد، لكنها **ليست** مستوى الحماية المستخدم في الأنظمة المؤسسية الكبيرة. إذا أردت مستقبلاً حماية أقوى (تسجيل دخول حقيقي متعدد المستخدمين، صلاحيات متدرجة)، يمكن ترقية النظام لاحقاً باستخدام Supabase Auth بدلاً من كلمة المرور الثابتة.

---

## 🛠️ كيف تعمل الصفحتان معاً؟

- أي شخص يفتح الرابط الرئيسي ويرسل ملاحظة → تُخزَّن فوراً في قاعدة بيانات Supabase
- أنت تفتح `/admin`، تُدخل كلمة المرور، وترى **كل** الملاحظات من جميع المستخدمين في الوقت الفعلي
- يمكنك تصفية الملاحظات حسب (عام/فصل محدد، نوع الملاحظة، الحالة)
- يمكنك تغيير حالة كل ملاحظة (جديد → تمت المراجعة → تم التطبيق / مرفوض) لتتبع ما طبّقته فعلياً في الإصدارات القادمة
